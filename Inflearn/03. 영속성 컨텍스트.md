# 영속성 컨텍스트
__JPA 에서 가장 중요한 것__

## 영속성 컨텍스트 (= 엔티티를 영구 저장하는 환경) 
```java
EntityManager.persist(entity);
```
- 엔티티매니저에서 persist() 메소드를 사용, 영속성 컨텍스트를 통해서 해당 엔티티를 영속성 시킨다.
- __영속성 컨텍스트__ 논리적인 개념이며 눈에 보이지 않는다.
- 우리는 엔티티매니저를 통해서 __영속성 컨텍스트__ 에 접근할 수 있다.

## 엔티티의 생명주기
- 비영속 (new/transient)
```java
// 객체를 생성, 비영속 상태
Member member = new Member();
member.setId(1L);
member.setName("홍길동");
```
  - JPA 와 관계가 없는 상태 (= 비영속)
- 영속 (managed)
```
Member member = new Member();
member.setId(1L);
member.setName("홍길동");


```
- 준영속 (detached)
- 삭제 (removed)

## 영속성 컨텍스트의 장점
- 1차 캐시
- 동일성(Identity) 보장
- 트랜잭션을 지원하는 쓰기 지연 (Transactional write-behind)
- 변경감지 (Dirty Checking)
- 지연로딩 (Lazy Loading)
